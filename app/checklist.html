Secure SDLC & Hardening Checklist for Agentic Systems

Canonical v1.0 · 114 controls · interactive reader

Pick the profiles your system matches to filter the list (Core applies to every system; profiles are additive). Click a control to see pass criteria, evidence, AGT risk, and dependency links. The floor badge marks non-waivable, launch-blocking controls. Click the status pill to mark Pass / Fail / Partial / N/A. Toggle Step order to list controls in implementation order (no-prerequisite controls first, then those that depend on them). Open a control to add a note / evidence reference. Statuses and notes live only in this tab — use Save file to download them as JSON, Load file to restore a saved file, or Export Excel to download the current filtered view as a styled workbook (nothing is stored automatically). Hover AGT / floor / profile tags for definitions; depends-on / affects chips jump to the related control.

Status values

Pass implemented, evidence exists · Fail missing/broken · Partial gaps remain · N/A not applicable (record reason) · Accepted Risk time-bound waiver (owner, expiry, compensating control) — not allowed for floor. The floor badge = Non-Waivable Release Floor: must Pass before launch, cannot be Accepted Risk.

Relationships and step order

Depends on = controls that must be in place first for this one to function/pass. Affects = the inverse (controls that rely on this one). Step order lists controls topologically: step 1 = no prerequisites (start here), each later step depends only on earlier ones.

Applicability profiles (additive: Core + matches)

Core every system · Tool-Using calls tools/APIs · RAG / Memory retrieval or persistent memory · Multi-Agent subagents/A2A · Regulated legal/residency/attestation · Cyber-Physical actuates physical/OT systems

Autonomy levels

A0 advisory · A1 assisted action · A2 supervised delegation · A3 autonomous operation

Risk model (AGT, used in the Risk column)

AGT-* are local IDs, mapped to OWASP Agentic T1-T17 and OWASP LLM Top 10 (LLM01-LLM10). ASI* = community naming, not a canonical OWASP identifier.
